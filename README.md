# Spatial-Analysis-Using-APIs


This project combines two API-based geospatial workflows to perform hydrological analysis using a watershed shapefile (W22). First, the NASA POWER workflow uses the shapefile to generate a grid of sample points within the watershed, retrieves daily meteorological variables (precipitation and temperature) from the NASA POWER API for each point, and aggregates them to produce a watershed-average daily time series, along with monthly and annual summaries and visualization plots. Second, the Daymet workflow follows a similar approach but retrieves high-resolution daily climate variables (e.g., precipitation, temperature, radiation) from the Daymet API for multiple points within the watershed and stores them in a combined CSV dataset. Together, these workflows demonstrate a complete geospatial data pipeline where a watershed boundary is converted into representative sampling locations, external climate APIs are queried, and the results are processed into scientifically meaningful hydrological metrics such as watershed-average precipitation, temperature trends, and temporal variability, making the framework directly applicable to ecohydrological modeling and RHESSys-based watershed analysis.
